# docs/

Project documentation lives here.

## Structure

| File / Folder | Contents |
|---|---|
| `architecture.md` | High-level system design and data-flow diagrams |
| `phases.md` | Roadmap — what each phase implements |
| `setup.md` | Developer setup guide (venv, .env, credentials) |
| `coding_standards.md` | Formatting, naming, type-hints, docstring conventions |

## Generating docs (future)

Sphinx or MkDocs can be configured to build HTML documentation from the
inline docstrings in each module.  This will be added in a later phase.
